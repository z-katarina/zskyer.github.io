Java面试复习（并发与多线程）

Java
  • 多线程
  • 并发
  • 面试

简单总结的Java多线程开发与并发编程相关面试题。针对有些细节问题，后面有时间的话再写专题分析吧。

实现多线程的两种方式

继承Thread类

实现Runnable接口

两种方式比较

  • Runnable方式可以避免Thread方式由于Java单继承特性而带来的局限性。
  • Runnable的代码可以被多个线程（Thread实例）共享，适合于多个线程处理同一资源的情况。

线程常用方法

  • sleep()：是Thread的静态方法，使一个正在运行的线程处于睡眠状态，但是不会释放对象锁，会抛出InterruptedException异常。
  • wait()：使一个线程处于等待（阻塞）状态，并且释放所持有的对象的锁，需在同步的代码块中调用。
  • yield()：停止当前线程，回到就绪状态，把执行机会让给相同或者更高优先级的线程。
  • join()：在当前线程中调用另一个线程的join()方法，则当前线程转入阻塞状态，直到另一个线程运行结束，当前线程再由阻塞转为就绪状态。
  • notify()：唤醒一个处于等待状态的线程，但并不能确切的唤醒某一个等待状态的线程，而是由JVM确定唤醒哪个线程，而且与优先级无关。
  • notifyAll()：唤醒所有处于等待状态的线程，但并不是给所有唤醒线程一个对象的锁，而是让它们竞争。

线程同步

volatile

  • volatile修饰的成员变量在每次被线程访问时，都是从主存中进行读取。
  • 在访问volatile变量时不会执行加锁操作，因此具有更高的效率。但是volatile变量只能确保可见性，不保证原子性，因此是线程不安全的。
  • 必须同时满足下面两个条件，才可以使用volatile变量：
    • 对变量的写操作不依赖于当前值。
    • 该变量没有包含在具有其他变量的不变式中。

synchronized

  • synchronized关键字可以将对象或者方法标记为同步，以实现对对象和方法的互斥访问。

  • 如果同一个方法内同时有两个或更多线程，则每个线程都有自己的局部变量拷贝。

  • 类的每个实例都有自己的对象级别锁。
  • 访问同一个类的不同实例对象中的同步代码块，不存在阻塞等待获取对象锁的问题。
  • 当一个线程访问synchronized代码块时，另一个线程仍然可以访问该对象的非synchronized代码块。
  • 一个线程访问synchronized代码块时，其他线程对object中所有其他synchronized代码块的访问都将被阻塞。

死锁

  • 死锁是两个或多个线程
  • 死锁产生的四个必要条件：
    • 互斥条件：一个资源每次只能被一个进程使用。
    • 请求与保持条件：一个进程因请求资源而阻塞时，对已获得的资源保持不放。
    • 不剥夺条件：进程已获得的资源，在未使用完之前，不能强行剥夺。
    • 循环等待条件：若干个进程之间形成一条头尾相接的循环等待资源关系。
  • 解决死锁：
    • 预防死锁：资源一次性分配、可剥夺资源、资源有序分配法
    • 避免死锁：银行家算法
    • 检测死锁：建立资源分配表和进程等待表
    • 解除死锁：剥夺资源、撤销进程
